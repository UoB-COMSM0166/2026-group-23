# Game v1.4 — 黑盒测试用例

> **范围**：`docs/Game_v1.4/`（Quantum Drop / 数字塔防）  
> **方法**：仅依据界面与可观察行为，不依赖阅读源码。  
> **入口**：浏览器打开 `index.html`（建议 Live Server）。

---

## 1. 测试环境

| 项 | 说明 |
|----|------|
| 浏览器 | Chrome / Edge 最新版 |
| 地址 | 本地静态服务打开 `docs/Game_v1.4/index.html` |
| 通过标准 | 无白屏、无卡死；控制台无未捕获异常（已知 warning 可记录） |
| 数据 | 当前进度主要为内存状态；刷新页面 = 新会话 |

---

## 2. 用例优先级

| 优先级 | 含义 |
|--------|------|
| **P0** | 阻塞发布：进游戏、战斗、胜败、结算与返回 |
| **P1** | 投球小游戏与波次节奏、多关卡解锁 |
| **P2** | 边界操作、资源加载降级 |

---

## 3. 用例表

### A. 启动页（launch）— P0

| ID | 优先级 | 场景 | 前置 | 步骤 | 预期结果 |
|----|--------|------|------|------|----------|
| L1 | P0 | 首次进入 | 无 | 打开 `index.html` | 出现启动画面（背景图或深色底 + 动效），画布正常 |
| L2 | P0 | 进入难度选择 | L1 | 等待画面就绪后按界面提示点击进入 | 进入难度选择界面（EASY / DIFFICULT） |
| L3 | P2 | 测试入口 | L1 | 若存在「测试」类按钮，点击 | 进入测试模式或说明文档描述的行为，不崩溃 |

### B. 难度选择（difficulty）— P0

| ID | 优先级 | 场景 | 前置 | 步骤 | 预期结果 |
|----|--------|------|------|------|----------|
| D1 | P0 | Easy | L2 | 选择 Easy | 进入关卡选择地图；后续局中起始资源/基地 HP 符合「简单」体验（相对宽松） |
| D2 | P0 | Difficult | L2 | 选择 Difficult | 进入关卡地图；相对 Easy 资源更紧或基地更脆（可对比 D1） |
| D3 | P1 | 难度切换 | 已从地图返回难度（若有） | 换难度再进关 | 新开局使用新难度，无混用 |

### C. 关卡地图（levelmap）— P0 / P1

| ID | 优先级 | 场景 | 前置 | 步骤 | 预期结果 |
|----|--------|------|------|------|----------|
| M1 | P0 | 首局仅第一关 | 新会话 + 选难度 | 观察地图节点 | 仅关卡 1 可进入；2–5 锁定或不可选 |
| M2 | P1 | 通关解锁 | 关卡 1 胜利 → STAGES 回地图 | 查看关卡 2 | 关卡 2 可进入 |
| M3 | P2 | 悬停信息 | 已解锁节点 | 鼠标划过节点 | 有关卡名/威胁等展示或高亮，无报错 |
| M4 | P0 | 进入战斗 | 点击已解锁关卡 | 进入 playing | HUD 显示金币/波次；金币与关卡说明大致一致（如关卡 1 起始较多） |

### D. 战斗 — 波次与投球小游戏 — P0 / P1

| ID | 优先级 | 场景 | 前置 | 步骤 | 预期结果 |
|----|--------|------|------|------|----------|
| W1 | P0 | 开局流程 | M4 | 新开局观察 | 在首波战斗前出现投球/小游戏环节（或设计文档规定顺序） |
| W2 | P1 | 小游戏进行中 | W1 | 小游戏未结束 | 不应提前进入下一波出怪（若设计为阻塞） |
| W3 | P0 | 小游戏结束 | 完成小游戏至结算结束 | 进入波次倒计时或本波战斗 |
| W4 | P1 | 波次间隔 | 清完一波 | 若有「波次结束确认」→ 点确定 | 再进入小游戏；无确认则直接进入小游戏 |
| W5 | P0 | 全胜 | 打完所有波、场上无怪 | 等待 | 出现胜利条件（如 MISSION COMPLETE / 进入结算） |

### E. 经济与建造 — P0

| ID | 优先级 | 场景 | 前置 | 步骤 | 预期结果 |
|----|--------|------|------|------|----------|
| E1 | P0 | 建造 | playing，金币足够 | 选塔 → 点在可建格 | 扣费并成功放置；路径格不可建 |
| E2 | P0 | 金币不足 | 选昂贵塔、金币不足 | 尝试放置 | 不放置或明确不可建，不出现异常扣费 |
| E3 | P1 | 击杀收益 | 击杀怪物 | 观察 HUD 金币 | 金币增加（定性即可） |

### F. 基地与失败 — P0

| ID | 优先级 | 场景 | 前置 | 步骤 | 预期结果 |
|----|--------|------|------|------|----------|
| F1 | P0 | 基地扣血 | playing | 放任怪物到达基地 | 基地 HP 下降 |
| F2 | P0 | 失败结算 | baseHp 归零 | 等待 | 出现失败界面（如 MISSION FAILED）；有 RETRY / STAGES |

### G. 结算面板（endpanel）— P0

| ID | 优先级 | 场景 | 前置 | 步骤 | 预期结果 |
|----|--------|------|------|------|----------|
| G1 | P0 | 重试 | 任意结算 | 点击 RETRY | 回到 playing，当前关卡重开，状态重置 |
| G2 | P0 | 返回选关 | 任意结算 | 点击 STAGES | 回到关卡地图 |
| G3 | P0 | 下一关 | 非最后一关胜利 | 点击 NEXT LEVEL（若显示） | 进入下一关卡战斗 |
| G4 | P1 | 最终关胜利 | 关卡 5 胜利 | 查看按钮 | 无错误 NEXT 或文案合理；RETRY/STAGES 仍可用 |

### H. 边界与鲁棒性 — P2

| ID | 优先级 | 场景 | 前置 | 步骤 | 预期结果 |
|----|--------|------|------|------|----------|
| B1 | P2 | 快速连点 | 结算/启动页 | 连续快速点击 | 不重复弹窗、阶段不错乱、不卡死 |
| B2 | P2 | 小游戏焦点 | 小游戏开启 | 尝试点击背后地图 | 仅小游戏响应，不误触建塔（若设计如此） |
| B3 | P2 | 窗口尺寸 | 任意阶段 | 改变浏览器窗口 | 画布仍可用或可接受留白（按课程要求记录） |

### I. 非功能 — P2
